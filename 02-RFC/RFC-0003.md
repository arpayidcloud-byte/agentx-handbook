# RFC-0003: Task Lifecycle State Machine
**Status:** Accepted | **Volume:** 2 (Core Runtime)

## Problem
Every task's execution state must be centrally inspectable and auditable, not scattered
across module-local state.

## Proposal
Single state machine (Queued → Planning → AwaitingApproval/Running → Completed/Failed/
Cancelled) owned by Core Runtime, with every transition publishing an event.

## Alternatives
- Per-module local state with polling — rejected, breaks auditability (Constitution
  Principle 7) and Event Driven principle (Principle 5).

## Decision
Adopted as specified in Volume 2, Ch. 1.
