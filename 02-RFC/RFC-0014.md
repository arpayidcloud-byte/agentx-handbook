# RFC-0014: v0.1 Deployment Topology
**Status:** Accepted | **Volume:** 11 (Cloud Platform)

## Problem
Designing a multi-service, horizontally-scaled topology now would be speculative — no
usage data exists to size it correctly.

## Proposal
Single-node deployment (all components on one machine/container) as the v0.1 default;
multi-service split deferred to a future RFC once Volume 10 usage data exists.

## Alternatives
- Design multi-service topology upfront — rejected, premature given current solo-developer
  usage pattern.

## Decision
Adopted as specified in Volume 11, Ch. 1.
