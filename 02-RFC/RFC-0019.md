# RFC-0019: Agent Output Evaluation Methodology
**Status:** Accepted | **Volume:** 14 (Testing & QA Strategy)

## Problem
Deterministic contract tests cannot validate whether LLM-generated code/output is
actually good — exact-match testing is meaningless for non-deterministic output.

## Proposal
Golden-set evaluation with rubric-based (not exact-match) scoring, run manually before
any agent prompt or default-provider change ships — not on every commit, to bound
provider cost.

## Alternatives
- Per-commit automated evaluation — rejected for v0.1, provider cost not yet justified by
  golden-set size.

## Decision
Adopted as specified in Volume 14, Ch. 2.
