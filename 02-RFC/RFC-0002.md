# RFC-0002: Event Bus Delivery Guarantee
**Status:** Accepted | **Volume:** 2 (Core Runtime)

## Problem
BullMQ/Redis-backed event delivery must choose between at-least-once and exactly-once
semantics; exactly-once requires a transactional outbox pattern with real implementation
cost.

## Proposal
At-least-once delivery; all consumers MUST be idempotent (dedupe on `event.id`).

## Alternatives
- Exactly-once via transactional outbox — rejected for v0.1: too costly for a
  solo-operator scope with no evidence of duplicate-event problems yet.

## Decision
Adopted as specified in Volume 2, Ch. 2. Revisit once Volume 13 telemetry shows real
duplicate-event rate. See ADR-0001.
