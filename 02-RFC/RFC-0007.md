# RFC-0007: Two-Layer Approval Gating
**Status:** Accepted | **Volume:** 5 (Workflow Engine)

## Problem
Tool-level destructive gating (Volume 7) alone cannot express process-level checkpoints
that aren't tied to a single dangerous tool call (e.g., "approve before auto-applying
Review Agent's suggestions").

## Proposal
A second, coarser `WorkflowPolicy` gate layer in Workflow Engine, additive to and never a
bypass of Tool SDK's gates.

## Alternatives
- Single unified gate inside Tool SDK only — rejected, conflates "inherently dangerous"
  with "requires process sign-off," two different concerns.

## Decision
Adopted as specified in Volume 5, Ch. 2.
