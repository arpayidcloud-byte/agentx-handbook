# RFC-0001: Provider Abstraction Design
**Status:** Accepted | **Volume:** 4 (Provider Platform)

## Problem
Core, Agent Platform, and Tool SDK must never depend on a specific LLM vendor's SDK
(Constitution Principle 3), but each vendor exposes completion and tool-calling APIs
differently.

## Proposal
A single `Provider` interface (Volume 4, Ch. 1) with a `NormalizedToolSpec`/
`NormalizedToolCall` shape (Ch. 2) that every vendor adapter translates to/from. Vendor
SDK imports are confined to `packages/provider-sdk/providers/*`.

## Alternatives
- Direct per-vendor integration in Agent Platform — rejected, violates Principle 3 directly.
- A generic "prompt string in, string out" interface with no normalized tool calls —
  rejected, loses structured tool-calling entirely, which Tool SDK depends on.

## Decision
Adopted as specified in Volume 4. See ADR-0003 for the two-adapter proof requirement.
