# RFC-0017: Metrics Derived From Existing Audit Data
**Status:** Accepted | **Volume:** 13 (Observability & SRE)

## Problem
Standing up a dedicated metrics/tracing service (e.g., Prometheus/OTel collector) before
v0.1 ships would add infrastructure Volume 1's exit criteria doesn't require.

## Proposal
All v0.1 metrics (Volume 13, Ch. 1) are computed from existing `AuditEvent`/`CostRecord`
data; `traceId` reconstruction via `agentx audit` substitutes for dedicated tracing
infrastructure.

## Alternatives
- Stand up Prometheus/Grafana/OTel now — rejected, premature infra before real query-
  latency problems are observed.

## Decision
Adopted as specified in Volume 13, Ch. 1–2.
