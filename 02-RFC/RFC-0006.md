# RFC-0006: Destructive Action Classification Default
**Status:** Accepted | **Volume:** 7 (Tool SDK)

## Problem
Distinguishing "safe" `fs.write` (new file) from "destructive" `fs.write` (overwrite)
precisely is more complex to implement correctly than a blanket rule, and a false
negative here risks silent data loss.

## Proposal
v0.1 classifies all `fs.write` calls as destructive (gated) regardless of new-vs-overwrite,
per Constitution Principle 7's fail-closed bias.

## Alternatives
- Precise new-vs-overwrite detection immediately — rejected for v0.1, deferred until
  classification logic has test coverage to justify relaxing it.

## Decision
Adopted as specified in Volume 7, Ch. 4 and Section 10. See ADR-0005.
