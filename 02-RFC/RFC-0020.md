# RFC-0020: Handbook Document Conventions
**Status:** Accepted | **Volume:** 1 (Foundation)

## Problem
Volumes, RFCs, and ADRs need a consistent status lifecycle, diagram notation, and
interface notation, or they drift into mutually inconsistent styles as the handbook grows
past one author's short-term memory.

## Proposal
Diffable inline Mermaid diagrams (no external image tools), TypeScript interface blocks
for contracts, and a single Draft → In Review → Approved/Accepted/Final → Superseded
lifecycle shared by all three document types.

## Alternatives
- External diagram tools (Figma/draw.io) — rejected, breaks version-control diffability
  and adds a second required tool, against the spirit of No Vendor Lock-in.

## Decision
Adopted as specified in Volume 1, Ch. 5.
