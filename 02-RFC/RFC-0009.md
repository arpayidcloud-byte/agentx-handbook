# RFC-0009: Plugin Extension Points & Manifest
**Status:** Accepted | **Volume:** 8 (Plugin Platform)

## Problem
New agents/tools/providers must be addable without modifying core packages
(Constitution Principle 4), but extension must not become a trust-escalation path.

## Proposal
Three typed extension points (agent/tool/provider plugins) with a manifest declaring
permissions upfront, reviewed once at install time; runtime enforcement stays identical
to built-in components (Volume 7's checks apply unchanged).

## Alternatives
- Untyped/arbitrary plugin hooks — rejected, no way to review permissions before install.

## Decision
Adopted as specified in Volume 8, Ch. 1–4.
