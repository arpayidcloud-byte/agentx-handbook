# RFC-0005: Fixed v0.1 Agent Roster
**Status:** Accepted | **Volume:** 3 (Agent Platform)

## Problem
An open-ended agent roster risks scope creep and an unclear trust boundary (which agent
may touch which tools).

## Proposal
Exactly 4 agents for v0.1: coding, review, test, security — matching prior project
scoping. A 5th agent requires a new RFC, not a config-only change.

## Alternatives
- Dynamic/open agent registration from day one — rejected, deferred to Plugin Platform
  (Volume 8) once the core roster is proven stable.

## Decision
Adopted as specified in Volume 3, Ch. 2. See ADR-0002.
