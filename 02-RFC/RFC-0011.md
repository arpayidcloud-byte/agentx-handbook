# RFC-0011: Multi-Tenancy Isolation Mechanism
**Status:** Accepted | **Volume:** 10 (Enterprise Platform)

## Problem
Application-layer tenant filtering alone has repeatedly proven insufficient in prior
project experience (a missing WHERE clause can leak cross-tenant data).

## Proposal
Defense-in-depth: Postgres Row-Level Security as the fail-closed database-layer control,
plus a tenant-scoped Prisma client extension at the application layer — neither alone is
sufficient.

## Alternatives
- Application-layer filtering only — rejected, directly repeats a known prior failure mode.

## Decision
Adopted as specified in Volume 10, Ch. 1. See ADR-0006.
