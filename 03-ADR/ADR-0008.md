# ADR-0008: No Push Alerting in v0.1
**Status:** Final | **Volume:** 13

## Context
Push alerting (email/Slack) requires a persistent server process, which v0.1's
single-node, CLI-driven topology (Volume 11) does not assume exists.

## Decision
v0.1 observability is pull-based only: `agentx status`/`watch`/`cost` (Volume 9) are the
notification surface. No external alerting channel ships in v0.1.

## Consequences
- Real issues can go unnoticed between active CLI sessions — an accepted, explicit
  trade-off, not an oversight.
- Push alerting is the natural Volume 10/11 follow-up once a persistent process exists.
