# ADR-0002: Fixed 4-Agent Roster for v0.1
**Status:** Final | **Implements:** RFC-0005 | **Volume:** 3

## Context
An open agent roster risks unclear trust boundaries and scope creep beyond the project's
committed v0.1 scope (3–4 agents, single-provider default).

## Decision
Ship exactly 4 agents in v0.1: coding, review, test, security. A 5th agent role requires
a new RFC, not a config change — this affects the trust boundary (tool access per role).

## Consequences
- Agent Registry (Volume 3, Ch. 3) is populated from static config, no dynamic
  registration UI needed for v0.1.
- Plugin Platform (Volume 8) cannot register an agent role colliding with this roster.
- Clear, auditable roadmap for adding agent #5+ later.
