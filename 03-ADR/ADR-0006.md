# ADR-0006: Postgres RLS + Tenant-Scoped Prisma Extension for Multi-Tenancy
**Status:** Final | **Implements:** RFC-0011 | **Volume:** 10

## Context
Application-layer-only tenant filtering has caused real multi-tenancy bugs in prior
project work — a single missing WHERE clause in one query path can leak cross-tenant data.

## Decision
Defense-in-depth: enable Postgres Row-Level Security on every tenant-scoped table AND use
a tenant-scoped Prisma client extension that fails closed if no tenant context is set.
Neither layer alone is sufficient by design.

## Consequences
- Every Volume 6 model requires an RLS policy before Volume 10 can be marked Approved.
- Mandatory cross-tenant-read contract test per model.
- Directly closes the exact gap this project has previously encountered.
