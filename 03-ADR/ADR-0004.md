# ADR-0004: Fail-Closed Permission Checks in Tool SDK
**Status:** Final | **Volume:** 7

## Context
Relying on prompt wording alone to keep an agent within its allowed tool categories has
proven insufficient in prior project experience — a model can be induced to attempt an
out-of-scope action regardless of its system prompt.

## Decision
Every tool execution passes through `PermissionChecker.isAllowed()` in code, checked
per-call, independent of what the prompt instructed. Denial throws before any I/O occurs.

## Consequences
- No tool implementation may bypass the check by calling execution logic directly —
  enforced via code review and a mandatory contract test (Volume 14).
- Slight per-call overhead, accepted as the cost of Constitution Principle 7 compliance.
