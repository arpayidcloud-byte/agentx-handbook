# ADR-0001: At-Least-Once Event Delivery on BullMQ/Redis
**Status:** Final | **Implements:** RFC-0002 | **Volume:** 2

## Context
Core Runtime needs an Event Bus (Constitution Principle 5). Exactly-once delivery over
Redis/BullMQ requires a transactional outbox pattern, adding real implementation cost
with no current evidence of a duplicate-event problem.

## Decision
Use at-least-once delivery. All event consumers must be idempotent, deduping on
`event.id`.

## Consequences
- Simpler v0.1 implementation, no outbox pattern needed.
- Every consumer (Memory Engine, Workflow Engine, CLI watch) must implement dedupe logic
  — a mandatory contract test requirement per Volume 14.
- Revisit once Volume 13 telemetry shows real duplicate rates.
