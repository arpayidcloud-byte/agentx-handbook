# ADR-0003: Two Working Provider Adapters Required at v0.1
**Status:** Final | **Implements:** RFC-0001 | **Volume:** 4

## Context
A single implemented provider adapter provides no real evidence the Provider Agnostic
abstraction (Constitution Principle 3) actually holds — an "abstraction of one."

## Decision
v0.1 must ship two full provider adapters (Anthropic and Google, matching the project's
existing toolchain) both passing an identical contract test suite against the `Provider`
interface, even though only one is the runtime default.

## Consequences
- More v0.1 implementation cost than a single-adapter approach.
- Concrete proof against silent provider coupling in Agent Platform or Tool SDK.
- Switching `DEFAULT_PROVIDER_ID` requires zero code changes outside Provider Platform.
